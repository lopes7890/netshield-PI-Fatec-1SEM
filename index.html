Empresa

A empresa Net-Shield tem como objetivo desenvolver um sistema de segurança da informação para proteger prontuários eletrônicos.

Com o aumento da digitalização dos registros médicos, a segurança desses dados se torna crucial para garantir a privacidade dos pacientes e a integridade das informações.

O Net-Shield implementará medidas de criptografia, autenticação e controle de acesso para assegurar que apenas pessoas autorizadas possam acessar os prontuários eletrônicos, além de monitorar atividades suspeitas e fornecer alertas em tempo real para prevenir possíveis violações de segurança.

Projeto

O Projeto Helix é uma solução de segurança da informação voltada à proteção de prontuários eletrônicos, utilizando a metodologia STRIDE para identificar e mitigar ameaças.

A proposta do sistema é garantir a confidencialidade, integridade e disponibilidade dos dados clínicos, que são altamente sensíveis. Para isso, o Helix implementa mecanismos como criptografia de dados, autenticação multifator e controle de acesso baseado em papéis (RBAC).

A modelagem STRIDE é aplicada para analisar possíveis vulnerabilidades no sistema, classificando ameaças em seis categorias. Com base nessa análise, o Helix define estratégias específicas de proteção para cada tipo de risco.

Assim, o Projeto Helix não apenas protege os dados médicos, mas também fortalece a confiança no sistema de saúde digital, garantindo que as informações dos pacientes estejam seguras contra acessos indevidos e ataques cibernéticos.

S

Spoofing — Falsificação de identidade

Ocorre quando alguém se passa por outra pessoa ou sistema para obter acesso indevido (ex: usar login e senha roubados).

T

Tampering — Violação de integridade

Refere-se à alteração indevida de dados, como modificar informações de um prontuário sem autorização.

R

Repudiation — Repúdio

Quando um usuário realiza uma ação e depois nega ter feito, dificultando a rastreabilidade.

I

Information Disclosure — Divulgação de informações

Exposição de dados sensíveis a pessoas não autorizadas, como vazamento de informações médicas.

D

Denial of Service — Negação de serviço

Ataques que tornam o sistema indisponível, impedindo o acesso aos prontuários.

E

Elevation of Privilege — Elevação de privilégio

Quando um usuário comum consegue acesso a funções de administrador ou níveis mais altos do sistema.
